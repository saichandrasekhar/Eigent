API Reference

Complete reference for the Eigent registry REST API. All endpoints accept and return JSON. Authentication is via OIDC bearer token or Eigent agent token.

Base URL: https://registry.eigent.dev/api (production) or http://localhost:3741/api (local)

Authentication: Include Authorization: Bearer <token> header on all requests.

POST /api/agents
Register a new agent and issue an Eigent token. The agent is bound to the authenticated human's identity.
ParameterTypeRequiredDescription
namestringrequiredUnique name for the agent
scopestring[]requiredList of allowed tool/action scopes
ttlstringoptionalToken time-to-live (e.g. "24h", "7d"). Default: "24h"
max_depthnumberoptionalMaximum delegation chain depth. Default: 5
metadataobjectoptionalArbitrary metadata to attach to the agent
Request
POST /api/agents
Content-Type: application/json
Authorization: Bearer <oidc-token>

{
  "name": "code-reviewer",
  "scope": ["read_file", "write_file", "run_tests"],
  "ttl": "24h",
  "max_depth": 5
}
Response 201 Created
{
  "id": "agt_x7k9m2p4",
  "name": "code-reviewer",
  "scope": ["read_file", "write_file", "run_tests"],
  "parent": "human:alice@company.com",
  "chain": ["human:alice@company.com"],
  "depth": 1,
  "max_depth": 5,
  "created_at": "2025-03-15T14:00:00Z",
  "expires_at": "2025-03-16T14:00:00Z",
  "token": "eyJhbGciOiJFUzI1NiIs..."
}
POST /api/agents/:id/delegate
Create a delegated sub-agent with narrowed permissions. The scope must be a subset of the parent agent's scope.
ParameterTypeRequiredDescription
namestringrequiredUnique name for the child agent
scopestring[]requiredScopes for child (must be subset of parent)
ttlstringoptionalToken TTL. Cannot exceed parent TTL.
Request
POST /api/agents/agt_x7k9m2p4/delegate
Content-Type: application/json
Authorization: Bearer <parent-agent-token>

{
  "name": "test-runner",
  "scope": ["run_tests"]
}
Response 201 Created
{
  "id": "agt_b3m8n1q7",
  "name": "test-runner",
  "scope": ["run_tests"],
  "parent": "agent:code-reviewer",
  "chain": [
    "human:alice@company.com",
    "agent:code-reviewer"
  ],
  "depth": 2,
  "token": "eyJhbGciOiJFUzI1NiIs..."
}
Error Response 400 Bad Request (scope violation)
{
  "error": "SCOPE_VIOLATION",
  "message": "Requested scope is not a subset of parent scope",
  "detail": {
    "parent_scope": ["read_file", "write_file", "run_tests"],
    "requested": ["deploy_prod"],
    "invalid": ["deploy_prod"]
  }
}
DELETE /api/agents/:id
Revoke an agent's token and cascade to all descendant agents. This is immediate and irreversible.
ParameterTypeRequiredDescription
cascadeboolean (query)optionalRevoke all descendants. Default: true
Request
DELETE /api/agents/agt_x7k9m2p4?cascade=true
Authorization: Bearer <human-oidc-token>
Response 200 OK
{
  "revoked": "agt_x7k9m2p4",
  "cascade_count": 2,
  "cascaded": [
    "agt_b3m8n1q7",
    "agt_c9p2k4w1"
  ],
  "audit_event": "evt_revoke_f8a2c1"
}
POST /api/verify
Verify whether an agent is authorized to perform a specific action. This is the same check the sidecar performs on every MCP tool call.
ParameterTypeRequiredDescription
agent_idstringrequiredAgent ID or name to verify
actionstringrequiredTool/action scope to check
resourcestringoptionalTarget resource for context-aware checks
Request
POST /api/verify
Content-Type: application/json

{
  "agent_id": "agt_b3m8n1q7",
  "action": "delete_file"
}
Response 200 OK
{
  "allowed": false,
  "agent": "test-runner",
  "action": "delete_file",
  "reason": "scope not granted in delegation chain",
  "available_scope": ["run_tests"],
  "chain": [
    "human:alice@company.com",
    "agent:code-reviewer",
    "agent:test-runner"
  ]
}
GET /api/audit
Query the Merkle-chained audit trail. Filter by agent, time range, event type, or chain. Returns events in reverse chronological order.
ParameterTypeRequiredDescription
agent_idstring (query)optionalFilter by agent ID
typestring (query)optionalEvent type: TOKEN_ISSUED, DELEGATION, SCOPE_ALLOWED, SCOPE_DENIED, REVOKED
fromISO 8601 (query)optionalStart timestamp
toISO 8601 (query)optionalEnd timestamp
limitnumber (query)optionalMax results. Default: 50, Max: 1000
verify_chainboolean (query)optionalVerify Merkle chain integrity. Default: false
Request
GET /api/audit?type=SCOPE_DENIED&limit=2&verify_chain=true
Authorization: Bearer <token>
Response 200 OK
{
  "events": [
    {
      "id": "evt_003_a3f2c1d8",
      "timestamp": "2025-03-15T14:32:01Z",
      "type": "SCOPE_DENIED",
      "agent": "test-runner",
      "action": "delete_file",
      "chain": ["alice", "code-reviewer", "test-runner"],
      "hash": "sha256:a3f2c1d8...",
      "prev_hash": "sha256:4e07408..."
    }
  ],
  "chain_valid": true,
  "total": 233
}
GET /api/.well-known/jwks.json
Retrieve the registry's JSON Web Key Set. Used by sidecars and external verifiers to validate Eigent token signatures. No authentication required.

GET /api/.well-known/jwks.json
Response 200 OK

  ]
}

Error Codes

All error responses follow this structure:

{
  "error": "ERROR_CODE",
  "message": "Human-readable description",
  "detail": { /* additional context */ }
}
CodeHTTPDescription
SCOPE_VIOLATION400Requested scope is not a subset of parent scope
DEPTH_EXCEEDED400Delegation would exceed max_depth
TOKEN_EXPIRED401Eigent token has expired
TOKEN_REVOKED401Eigent token has been revoked
INVALID_SIGNATURE401Token signature verification failed
CHAIN_BROKEN401Delegation chain is invalid or contains a revoked ancestor
EIGENT_SCOPE_DENIED403Action not in agent's effective scope
AGENT_NOT_FOUND404Agent ID does not exist in registry
DUPLICATE_AGENT409Agent with this name already exists
