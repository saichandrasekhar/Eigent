Quick Start

From zero to enforced delegation chains in 5 minutes. No infrastructure required — Eigent runs locally with a built-in registry.

Prerequisites

Node.js 18+ Runtime for CLI and registry
npm or yarn Package manager
Claude Desktop optional For MCP sidecar integration

Install the CLI

$ npm install -g @eigent/cli

added 42 packages in 3s
1 package is looking for funding

# Verify installation
$ eigent --version
eigent v0.3.0

Initialize a local registry

This creates a local Eigent registry with a fresh ECDSA keypair for signing tokens. In production, you'd point to a shared registry.

$ eigent init

✅ Registry initialized at ~/.eigent/
   Keypair:  ES256 (P-256 ECDSA)
   JWKS:     ~/.eigent/jwks.json
   Database: ~/.eigent/registry.db
   Config:   ~/.eigent/config.yaml

Authenticate (establish the root of trust)

Login establishes your human identity as the root of the delegation chain. In dev mode, this uses a local identity. In production, use OIDC.

# Development mode (local identity)
$ eigent login

✅ Authenticated as dev@localhost
   Mode: local development
   Session expires: 8 hours

# Production mode (OIDC)
$ eigent login --provider okta --domain company.okta.com

✅ Authenticated as alice@company.com
   Provider: Okta

Issue a token for an agent

Create a cryptographically signed token that binds the agent's identity to yours. Specify exactly which scopes (tool permissions) the agent is allowed.

$ eigent issue code-reviewer \
    --scope read_file,run_tests \
    --ttl 24h

✅ Token issued for agent "code-reviewer"
   Agent ID:  agt_x7k9m2p4
   Scopes:    read_file, run_tests
   Bound to:  dev@localhost
   Chain:     dev@localhost → code-reviewer
   Depth:     1/5
   Expires:   2025-03-16T14:00:00Z
   Token:     eyJhbGciOiJFUzI1NiIs...

Delegate to a sub-agent (with narrowed scope)

Agent code-reviewer can delegate to test-runner, but only with a subset of its own permissions. You cannot grant more than you have.

$ eigent delegate code-reviewer test-runner --scope run_tests

✅ Delegated "test-runner" from "code-reviewer"
   Scopes:    run_tests
   Narrowed:  [read_file, run_tests] → [run_tests]
   Chain:     dev@localhost → code-reviewer → test-runner
   Depth:     2/5

Try granting a scope the parent doesn't have:

$ eigent delegate code-reviewer deployer --scope deploy_prod

❌ ERROR: Cannot delegate scope "deploy_prod"
   Parent "code-reviewer" only has: [read_file, run_tests]
   Requested "deploy_prod" is not a subset

Verify permissions

Check whether an agent is allowed to perform a specific action. This is what the sidecar does automatically on every tool call.

# Can test-runner delete files?
$ eigent verify test-runner delete_file

❌ DENIED
   Agent:     test-runner
   Requested: delete_file
   Available: [run_tests]
   Chain:     dev@localhost → code-reviewer → test-runner

# Can test-runner run tests?
$ eigent verify test-runner run_tests

✅ ALLOWED
   Agent:     test-runner
   Scope:     run_tests
   Chain:     dev@localhost → code-reviewer → test-runner

Wrap your MCP server with the Eigent sidecar

Add the Eigent sidecar to your Claude Desktop configuration. The sidecar sits between Claude and your MCP server, enforcing permissions on every tool call.

Update your claude_desktop_config.json:

{
  "mcpServers": {
    "filesystem": {
      "command": "eigent",
      "args": [
        "sidecar",
        "--agent", "code-reviewer",
        "--wrap", "npx",
        "--",
        "@modelcontextprotocol/server-filesystem",
        "/Users/alice/projects"
      ]
    }
  }
}

Before (no Eigent):

// Direct MCP access — no identity, no scope, no audit
{
  "command": "npx",
  "args": ["@modelcontextprotocol/server-filesystem", "/Users/alice/projects"]
}

After (with Eigent):

// Eigent sidecar wraps MCP — identity bound, scoped, audited
{
  "command": "eigent",
  "args": ["sidecar", "--agent", "code-reviewer", "--wrap", "npx",
           "--", "@modelcontextprotocol/server-filesystem", "/Users/alice/projects"]
}
