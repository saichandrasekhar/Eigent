Step-by-Step Flow

Follow a request from human authentication through delegation, enforcement, and audit.

Human Authenticates via OIDC

The human operator authenticates through their organization's identity provider (Okta, Entra ID, or any OIDC-compliant IdP). This establishes the root of trust for the entire delegation chain.

// OIDC authentication flow
$ eigent login --provider okta

// Opens browser for OIDC authentication
✅ Authenticated as alice@company.com
   Provider: Okta (company.okta.com)
   Subject:  00u1a2b3c4d5e6f7g8h9
   Session:  expires in 8 hours

Human Issues Eigent Token for Agent A

The human creates a cryptographically signed token (JWS) that binds an agent identity to their own. The token contains the agent's allowed scopes, its parent (the human), and the delegation chain metadata.

$ eigent issue code-reviewer \
    --scope read_file,write_file,run_tests \
    --ttl 24h

The JWS token structure:

// Header
{
  "alg": "ES256",
  "typ": "eigent+jwt",
  "kid": "reg-key-001"
}

// Payload
{
  "sub":       "agent:code-reviewer",
  "iss":       "https://registry.eigent.dev",
  "iat":       1711900800,
  "exp":       1711987200,
  "scope":     ["read_file", "write_file", "run_tests"],
  "parent":    "human:alice@company.com",
  "chain":     ["human:alice@company.com"],
  "depth":     1,
  "max_depth": 5
}

// Signature: ES256 over header.payload

Agent A Delegates to Agent B (Permission Narrowing)

Agent A creates a sub-agent with a subset of its own permissions. The scope can only narrow — never widen. The delegation chain grows by one hop, and the depth counter increments.

$ eigent delegate code-reviewer test-runner --scope run_tests

✅ Delegated "test-runner" from "code-reviewer"
   Scopes: run_tests
   Narrowed: [read_file, write_file, run_tests] → [run_tests]
   Chain: alice → code-reviewer → test-runner
   Depth: 2/5
PERMISSION NARROWING
Alice (human) read, write, test, deploy
code-reviewer read, write, test
test-runner test

Sidecar Intercepts MCP Traffic

The Eigent sidecar proxy sits between the AI agent and the MCP server. Every JSON-RPC call is intercepted. The sidecar validates the token signature, checks the delegation chain, and performs a 3-way scope intersection before forwarding the request.

// Sidecar validation pipeline

1. Extract eigent token from request headers
2. Verify JWS signature against registry JWKS
3. Validate token expiry and delegation chain
4. Compute scope intersection:
   effective_scope = human_perms ∩ agent_scope ∩ resource_sensitivity
5. Check requested tool against effective_scope
6. ALLOW or DENY the request
7. Log decision to audit trail

Unauthorized Call BLOCKED

When test-runner tries to call delete_file, the sidecar rejects the request because delete_file is not in the agent's effective scope. The response includes the full denial reason and chain context.

// JSON-RPC error response from sidecar
{
  "jsonrpc": "2.0",
  "id": 42,
  "error": {
    "code": -32001,
    "message": "EIGENT_SCOPE_DENIED",
    "data": {
      "agent":     "test-runner",
      "requested": "delete_file",
      "available": ["run_tests"],
      "chain":     [
        "human:alice@company.com",
        "agent:code-reviewer",
        "agent:test-runner"
      ],
      "reason": "scope 'delete_file' not granted in delegation chain"
    }
  }
}

Everything Logged to Audit Trail

Every action — token issuance, delegation, verification, allow, deny — is recorded as a Merkle-chained event. Each entry includes a hash of the previous entry, making the log tamper-evident. The audit trail maps directly to EU AI Act and SOC2 controls.

// Audit trail entry (Merkle-chained)
{
  "id":        "evt_a8f3c2d1",
  "timestamp": "2025-03-15T14:32:01Z",
  "type":      "SCOPE_DENIED",
  "agent":     "test-runner",
  "action":    "delete_file",
  "chain":     ["alice", "code-reviewer", "test-runner"],
  "prev_hash": "sha256:9f86d08...",
  "hash":      "sha256:4e07408...",
  "compliance": {
    "eu_ai_act": ["Art.9", "Art.12"],
    "soc2":      ["CC6.1", "CC7.2"]
  }
}

What Makes This Different

Traditional IAM authenticates users and machines. Eigent solves the harder problem: multi-hop agent delegation.

● Not Just Authentication

OAuth authenticates users. Eigent authenticates the chain of decisions that led to an agent having its permissions.

● Not Just Authorization

RBAC assigns roles. Eigent enforces that permissions can only decrease as they flow through agent-to-agent delegation.

● Not Just Logging

SIEM collects events. Eigent creates a Merkle-chained audit trail that cryptographically proves no events were tampered with or deleted.
