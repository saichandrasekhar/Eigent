FAQ

Frequently asked questions

Everything you need to know about AI agent security and Eigent.

MCP (Model Context Protocol) server security refers to the practice of discovering, auditing, and securing MCP servers that AI agents use to access tools, data, and external services. Without proper security, MCP servers can expose sensitive data, allow unauthorized tool execution, or create unmonitored shadow agent infrastructure. Eigent Scan automatically discovers all MCP servers in your environment and flags misconfigurations, missing authentication, and other security risks.
Shadow AI agents are autonomous agents running in your environment without IT or security team knowledge. Eigent detects them by scanning MCP configuration files across all known clients (Claude Desktop, Cursor, VS Code, etc.), monitoring running processes for stdio and SSE transports, and identifying unregistered MCP servers. Run eigent-scan scan --verbose to get a complete inventory of every agent and MCP server in your environment.
Yes, Eigent is completely free and open source under the Apache 2.0 license. Eigent Scan (discovery) and Eigent Sidecar (observability) are available today at no cost. The upcoming Eigent Platform (governance dashboard) will have both a free community tier and enterprise tiers for organizations that need advanced compliance features.
The EU AI Act is the world's first comprehensive AI regulation, requiring organizations to maintain transparency, accountability, and risk management for AI systems. For autonomous AI agents, this means you need an inventory of all agents, audit trails of their actions, and proof of governance. Eigent helps you meet these requirements by automatically discovering agents, capturing complete audit logs via the Sidecar, and generating compliance reports aligned with EU AI Act, SOC2, and ISO 27001 frameworks.
While Astrix Security focuses on non-human identity management for API keys and service accounts, Eigent is purpose-built for AI agent trust infrastructure. Eigent specifically addresses MCP server discovery, agent-to-tool call tracing, delegation chain capture, and AI-specific compliance requirements. Eigent is also fully open source (Apache 2.0), giving you complete transparency and no vendor lock-in.
Yes. Eigent Sidecar is OpenTelemetry native, meaning it exports traces and metrics in the standard OTLP format. You can route agent activity data to any SIEM or observability platform that supports OpenTelemetry, including Splunk, Datadog, Elastic, Grafana, and others. This lets you correlate AI agent behavior with your existing security monitoring without changing your toolchain.
Getting started takes less than 60 seconds. Install with pip install eigent-scan, then run eigent-scan scan --verbose to discover every MCP server in your environment. Eigent Scan checks Claude Desktop, Cursor, VS Code, and running processes automatically. No configuration needed. Use eigent-scan report --format html to generate a shareable security report.
Eigent works with any AI agent that uses the Model Context Protocol (MCP), which is the emerging standard for agent-tool communication. This includes agents built with Claude, GPT, and other LLMs that connect to MCP servers. The Sidecar intercepts MCP traffic at the transport layer (stdio and SSE), so it works regardless of which framework or LLM the agent uses.
