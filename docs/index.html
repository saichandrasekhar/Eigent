Core Capabilities

Four pillars of agent trust

Every autonomous agent needs a cryptographic identity, bounded permissions, runtime enforcement, and a tamper-proof audit trail.

🔐

Identity Binding

Every agent gets an Ed25519 JWS token bound to an OIDC-verified human (Okta, Entra ID, Google). SCIM deprovisioning auto-revokes when users leave. Multi-tenancy isolates organizations.

🔗

Delegation Chains

Agents delegate to sub-agents with mandatory permission narrowing. Three-way scope intersection ensures least privilege. Cascade revocation kills entire trust subtrees instantly.

🛡

Policy Engine + Enforcement

MCP sidecar (stdio + HTTP) with YAML policy engine: glob patterns, argument regex, time windows, depth limits. Approval queue for sensitive ops. OTel spans and Prometheus metrics.

📑

Compliance + Audit

Complete audit trail with SIEM webhooks (Splunk, Datadog, Slack). Compliance reports for EU AI Act and SOC 2. Python SDK, 16-command CLI, Next.js dashboard with SSO and RBAC.

See It In Action

From identity to enforcement in 4 commands

Issue a token, delegate with narrowed scope, and watch unauthorized actions get blocked in real time.

eigent-demo — zsh
# Issue a token for code-reviewer agent with specific scopes
$ eigent issue code-reviewer --scope read_file,write_file,run_tests
✅ Token issued for agent "code-reviewer"
   Scopes: read_file, write_file, run_tests
   Bound to: alice@company.com (OIDC verified)
   Chain depth: 1/5

# Delegate to sub-agent with narrowed permissions
$ eigent delegate code-reviewer test-runner --scope run_tests
✅ Delegated "test-runner" from "code-reviewer"
   Scopes: run_tests (narrowed from 3 → 1)
   Chain: alice → code-reviewer → test-runner
   Chain depth: 2/5

# Verify: can test-runner delete files?
$ eigent verify test-runner delete_file
❌ DENIED — scope "delete_file" not in agent permissions [run_tests]
   Chain: alice → code-reviewer → test-runner
   Required: delete_file | Available: run_tests

# Verify: can test-runner run tests?
$ eigent verify test-runner run_tests
✅ ALLOWED — scope "run_tests" verified through delegation chain
   Chain: alice → code-reviewer → test-runner
